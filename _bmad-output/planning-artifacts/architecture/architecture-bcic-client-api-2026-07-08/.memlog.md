- (decision) Use a ports-and-adapters/hexagonal SDK architecture: public Client and endpoint modules depend inward on ports and models; auth and transport are adapters behind internal interfaces.
- (decision) Public API is domain-first: callers use Client endpoint properties and typed methods; raw REST v1 method calls exist only as a documented lower-level escape hatch.
- (decision) REST v1 transport owns /rest/api/{method_name} URL construction, output format handling, response normalization, retry policy, and SDK exception mapping; endpoint modules never construct HTTP requests directly.
- (decision) Authentication is a strategy owned by auth.py and attached by transport; endpoint modules never accept or concatenate session IDs.
- (decision) All external failures are mapped into the SDK exception hierarchy before crossing the public API boundary; httpx and parser exceptions do not leak to callers.
- (decision) Pagination is normalized behind page/list_all helpers; endpoint modules expose pagination capabilities but do not each implement page traversal loops.
